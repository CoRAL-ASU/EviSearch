# Accuracy Report – All Trials

This folder is generated by `aggregate_accuracy_report.py`. Re-run from the repo root to refresh after new evaluations:

```bash
python new_pipeline_outputs/aggregate_accuracy_report.py
# or with venv:
.venv/bin/python new_pipeline_outputs/aggregate_accuracy_report.py
```

## Outputs

| File | Description |
|------|-------------|
| **accuracy_report.xlsx** | Excel workbook: **Summary** (one row per trial: avg overall, correctness, completeness), **Per-Column Scores** (rows = columns, one column per trial; red = wrong &lt; 1.0, green = correct), **Wrong Only** (all trial–column pairs with score &lt; 1.0 and GT/predicted/reason). |
| **accuracy_report.html** | Same summary + per-column table in one HTML page; open in a browser. Red/green cells for wrong/correct. |
| **accuracy_per_trial.png** | Bar chart: average overall score per trial (green ≥ 0.7, red &lt; 0.5, orange in between). |
| **heatmap_per_column_trial.png** | Heatmap: rows = columns, columns = trials; color = score (green = correct, red = wrong). |

Trials are discovered automatically from `new_pipeline_outputs/results`: any trial folder that has an `evaluation/evaluation_results.json` (or under `latest/` or `run_*/`) is included.
